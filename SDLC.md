# SDLC 계획

## 전체 흐름

```
요구사항 정의 → 설계 → 개발(MVP) → 테스트 → 배포 v1 → 운영 → 배포 v2
   ✅           🔄        🔄          🔄 ◀현재      ⬜        ⬜        ⬜
```

**현재 위치 (2026-07-18)**: 5단계(실기기 테스트) 진행 중. 핵심 마일스톤인 **러닝→토큰 연동을 실기기에서 검증 완료**(백그라운드 `(:background)` 버그 해결, DEVLOG 2026-07-17~18 참고). 앞 단계(2~4)는 대체로 끝났으나 캐릭터 아트 리뉴얼·밸런스 미세조정·디버그 코드 정리가 배포 전 과제로 남음.

---

## 단계별 상세

### 1단계 · 요구사항 정의
- **내용**: 게임 루프, 성장/상태 조건, 토큰 시스템, MVP 범위 확정
- **결과물**: 요구사항 문서
- **상태**: ✅ 완료

### 2단계 · 설계
- **내용**: 캐릭터 픽셀 디자인, 상태별 스프라이트 계획, 데이터 저장 구조, 화면 레이아웃
- **결과물**: 디자인 시안, 구조 설계서
- **상태**: 🔄 대체로 완료 (저장 구조·레이아웃·데이터 읽기/알림 지원 확인 끝). **캐릭터 아트 리뉴얼 진행 예정** — 더 귀엽게 + 검은 배경 대비용 밝은 외곽선 + 노란 스카프 시그니처 (REQUIREMENTS.md 디자인 브리프 참고)
- **비고**:
  - Aseprite 등 픽셀 아트 툴 사용 (실제로는 PixelLab 사용 중)
  - 성장 3단계 × 정상/아픈 2상태 = 최소 6종 스프라이트
  - ✅ Connect IQ 달리기 데이터 읽기 가능 확인 (실기기 검증 완료, 2026-07-18)
  - ✅ Connect IQ 알림 기능: `Background.requestApplicationWake()`로 대체 구현

### 3단계 · 개발 · MVP
- **내용**: 토큰 시스템, 밥주기 인터랙션, 시간 기반 성장, 상태(정상/아픈/사망) 구현
- **결과물**: Connect IQ `.iq` 파일
- **상태**: 🔄 핵심 시스템 구현 완료. 배포 전 정리 남음 — 디버그 스캐폴딩 제거, `tokens=15` 시작값 핵 제거, 포그라운드 `onStart` tick() 간헐 크래시 근본 원인
- **비고**: Monkey C 언어

### 4단계 · 밸런스 조정
- **내용**: 토큰 획득량, 밥 소모량, 성장 시간 기준 수치 조정
- **결과물**: 밸런스 확정 수치
- **상태**: 🔄 진행 중 (가안 수치로 구현됨, 실기기 플레이로 미세조정 예정)
- **비고**: 시뮬레이터 및 실기기 플레이 테스트 기반

### 5단계 · 테스트 ◀ **현재**
- **내용**: 시뮬레이터 → Forerunner 255 직접 테스트
- **결과물**: 테스트 결과 보고
- **상태**: 🔄 진행 중. **러닝→토큰 연동 실기기 검증 완료**(백그라운드 서비스 버그 해결, 2026-07-18). 남은 검증: 실제 야외 러닝으로 최종 확인(현재까지는 제자리 5분 러닝 테스트로만 확인)
- **비고**: USB로 직접 설치하며 확인

### 6단계 · 배포 v1
- **내용**: Connect IQ 스토어 등록, 사용자 피드백 수집
- **결과물**: 스토어 등록 완료
- **비고**: 심사 수일 소요

### 7단계 · 운영
- **내용**: 버그 수정, 꾸미기/캐릭터 선택 설계, v2 기획
- **결과물**: v2 요구사항 문서
- **비고**: 피드백 반영

### 8단계 · 배포 v2
- **내용**: 꾸미기 아이템, 캐릭터 종류 선택, 멀티 기기 지원
- **결과물**: 업데이트 배포
- **비고**: 반복 사이클
