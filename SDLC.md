# SDLC 계획

## 전체 흐름

```
요구사항 정의 → 설계 → 개발(MVP) → 테스트 → 배포 v1 → 운영 → 배포 v2
```

---

## 단계별 상세

### 1단계 · 요구사항 정의
- **내용**: 게임 루프, 성장/상태 조건, 토큰 시스템, MVP 범위 확정
- **결과물**: 요구사항 문서
- **상태**: ✅ 완료

### 2단계 · 설계
- **내용**: 캐릭터 픽셀 디자인, 상태별 스프라이트 계획, 데이터 저장 구조, 화면 레이아웃
- **결과물**: 디자인 시안, 구조 설계서
- **비고**:
  - Aseprite 등 픽셀 아트 툴 사용
  - 성장 3단계 × 정상/아픈 2상태 = 최소 6종 스프라이트
  - Connect IQ 달리기 데이터 읽기 가능 여부 사전 확인 필요
  - Connect IQ 알림 기능 지원 여부 확인 필요

### 3단계 · 개발 · MVP
- **내용**: 토큰 시스템, 밥주기 인터랙션, 시간 기반 성장, 상태(정상/아픈/사망) 구현
- **결과물**: Connect IQ `.iq` 파일
- **비고**: Monkey C 언어

### 4단계 · 밸런스 조정
- **내용**: 토큰 획득량, 밥 소모량, 성장 시간 기준 수치 조정
- **결과물**: 밸런스 확정 수치
- **비고**: 시뮬레이터 및 실기기 플레이 테스트 기반

### 5단계 · 테스트
- **내용**: 시뮬레이터 → Forerunner 255 직접 테스트
- **결과물**: 테스트 결과 보고
- **비고**: USB로 직접 설치하며 확인

### 6단계 · 배포 v1
- **내용**: Connect IQ 스토어 등록, 사용자 피드백 수집
- **결과물**: 스토어 등록 완료
- **비고**: 심사 수일 소요

### 7단계 · 운영
- **내용**: 버그 수정, 꾸미기/캐릭터 선택 설계, v2 기획
- **결과물**: v2 요구사항 문서
- **비고**: 피드백 반영

### 8단계 · 배포 v2
- **내용**: 꾸미기 아이템, 캐릭터 종류 선택, 멀티 기기 지원
- **결과물**: 업데이트 배포
- **비고**: 반복 사이클
