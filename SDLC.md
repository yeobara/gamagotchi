# SDLC 계획

## 전체 흐름

```
요구사항 정의 → 설계 → 개발(MVP) → 테스트 → 배포 v1 → 운영 → 배포 v2
```

---

## 단계별 상세

### 1단계 · 요구사항 정의
- **내용**: 성장 조건(km 기준), 화면 구성, MVP 범위 확정
- **결과물**: 요구사항 문서
- **상태**: ✅ 완료

### 2단계 · 설계
- **내용**: 캐릭터 픽셀 디자인, 데이터 저장 구조, 화면 레이아웃
- **결과물**: 디자인 시안, 구조 설계서
- **비고**: Aseprite 등 픽셀 아트 툴 사용

### 3단계 · 개발 · MVP
- **내용**: km 읽기, 알/아기/어른 구현, 워치페이스 표시
- **결과물**: Connect IQ `.iq` 파일
- **비고**: Monkey C 언어

### 4단계 · 테스트
- **내용**: 시뮬레이터 → Forerunner 255 직접 테스트
- **결과물**: 테스트 결과 보고
- **비고**: USB로 직접 설치하며 확인

### 5단계 · 배포 v1
- **내용**: Connect IQ 스토어 등록, 사용자 피드백 수집
- **결과물**: 스토어 등록 완료
- **비고**: 심사 수일 소요

### 6단계 · 운영
- **내용**: 버그 수정, 밥주기 인터랙션 설계, v2 기획
- **결과물**: v2 요구사항 문서
- **비고**: 피드백 반영

### 7단계 · 배포 v2
- **내용**: 밥주기, 멀티 기기 지원, 추가 성장 단계
- **결과물**: 업데이트 배포
- **비고**: 반복 사이클
