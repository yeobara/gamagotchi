# 요구사항 정의

## 핵심 게임 루프

```
달리기 → 토큰 획득 → 밥주기 → 캐릭터 생존 유지
                              ↑
                    48h 미급식 → 아픈 상태
                                    ↓ 추가 24h
                               사망 → 리셋 (알부터)
```

- 성장은 **시간 기반** 자동 진화 (달리기와 무관)
- 달리기는 **토큰**을 획득하는 수단
- 토큰은 밥주기 / 꾸미기에 사용
- 밥을 주지 않으면 패널티 → 긴장감 유지

---

## 캐릭터 상태

| 상태 | 조건 | 비고 |
|------|------|------|
| 정상 | 최근 48시간 내 밥주기 완료 | 성장 단계별 정상 모습 |
| 아픈 상태 | 마지막 밥주기로부터 48시간 경과 | 각 성장 단계별 아픈 버전 |
| 사망(리셋) | 아픈 상태에서 추가 24시간 경과 | 알부터 다시 시작 |

---

## 캐릭터

**v1 기본 캐릭터: 펭귄**
- 원래 못 달리는 펭귄이 달리기로 성장한다는 서사
- 귀여움 최우선
- v1 출시 후 반응 좋으면 v2에서 다른 캐릭터 추가

## 성장 단계

| 단계 | 조건 | 캐릭터 | 모습 |
|------|------|--------|------|
| 알 | 시작 | 펭귄 알 | 흰색 동글동글한 알 |
| 아기 | 시간 경과 (자동) | 아기 펭귄 | 회색 솜털, 동그란 눈 |
| 어른 | 시간 경과 (자동) | 어른 펭귄 | 블랙&화이트, 자신감 넘치는 포즈 |

> 성장 조건(시간 기준)은 밸런스 조정 단계에서 확정

---

## 토큰 시스템

- 달리기(러닝 활동만 카운트)로 토큰 획득
- 토큰 사용처: 밥주기, 꾸미기 아이템
- 토큰 획득량 / 밥 소모량 등 수치 밸런스는 추후 조정

---

## 픽셀 아트 리소스

각 성장 단계 × 2가지 상태(정상/아픈) = 최소 6종 스프라이트 필요

| 성장 단계 | 정상 | 아픈 상태 |
|-----------|------|-----------|
| 펭귄 알 | ✅ 필요 | ✅ 필요 |
| 아기 펭귄 | ✅ 필요 | ✅ 필요 |
| 어른 펭귄 | ✅ 필요 | ✅ 필요 |

- 아픈 상태: 눈이 풀리고 땀 흘리는 버전
- 픽셀 아트 직접 제작 (Aseprite 등), 32×32px 권장
- 애니메이션: 캐릭터당 **2프레임** 루프 (예: 날개 올림 ↔ 내림). 2프레임으로도 살아있는 느낌 충분, 배터리·메모리 부담 최소화
- `Toybox.Timer`로 프레임 교체, 손목 들었을 때 애니메이션 시작

---

## 화면 구성

대상 기기: Forerunner 255 (260×260px 원형, MIP 디스플레이 64색)

- 캐릭터: 화면 중앙, 크게 배치
- 현재 시간: 상단
- 누적 km / 보유 토큰: 하단
- 손목을 들었을 때 2~3초 애니메이션 재생

---

## 디자인 원칙

1. **귀여움 최우선** — 모든 디자인 결정의 1순위
2. **딱 보여야 함** — 화면을 보는 순간 캐릭터 상태를 즉시 알 수 있어야 함
3. **자랑하고 싶어야 함** — 손목 사진을 찍어 공유하고 싶을 만큼 귀여워야 함
4. **조작 최소화** — 달리기 외에 별도 노력 최소. 앱이 알아서 돌아가야 함
5. **감정 연결** — 캐릭터가 아프면 걱정되고, 성장하면 뿌듯한 느낌
6. **시계 기능 포함** — 시간이 항상 보여야 함

## MVP 범위

### v1 (MVP)
- 런 중 거리 읽기 → 토큰 변환 → Storage 저장
- 시간 기반 자동 성장 (3단계)
- 밥주기 인터랙션 (버튼)
- 아픈 상태 / 사망(리셋) 구현
- 장치 앱 기본 화면 (시간 + 캐릭터 + 토큰)

### v2 (업데이트)
- 꾸미기 아이템 시스템
- 캐릭터 추가 (v1 반응 좋으면 — 펭귄 외 다른 캐릭터)
- 멀티 기기 지원

## 앱 타입

**장치 앱 (Device App)**
- 워치페이스는 버튼 인터랙션 불가 확인 → 장치 앱으로 결정
- 버튼으로 밥주기 등 인터랙션 구현
- 시간 표시 직접 구현 필요

## 기술 제약 및 확인 필요 사항

### 확인 완료 ✅

- **런 중 거리 읽기** — `Activity.getActivityInfo().elapsedDistance`로 현재 달리기 거리(미터) 실시간 읽기 가능
  - 누적 히스토리 API는 없음 → 앱이 열려 있는 동안 거리를 읽고 직접 저장하는 방식으로 구현
- **데이터 저장** — `Application.Storage.setValue/getValue`로 영구 저장 가능 (앱 재시작 후에도 유지)
  - 저장 가능 타입: Number, Float, String, Boolean, Array, Dictionary 등
  - 값당 최대 32KB 제한
- **토큰 계산** — API 문제 아님. 저장된 거리를 수식으로 토큰으로 변환하면 됨 (예: 1km = N토큰)

### 핵심 데이터 흐름

```
앱 열림 (런 중) → elapsedDistance 읽기 → 런 종료 시 거리 저장
→ 거리 × 토큰 공식 = 토큰 획득 → Storage에 누적 저장
```

### 필수 확인 (개발 시작 전)

- **[확인 완료 ✅] 백그라운드 런 감지** — `Background.registerForActivityCompletedEvent()`로 앱이 닫혀 있어도 런 종료 이벤트 수신 가능
  - `ServiceDelegate`를 상속한 백그라운드 서비스 클래스 구현 → `onActivityCompleted()` 메서드에서 거리 읽기 → 토큰 계산 → `Storage` 저장 → `Background.exit(data)`로 메인 앱에 전달
  - 메인 앱 열릴 때 `AppBase.onBackgroundData()`로 백그라운드 결과 수신
  - 관련 API: `Toybox.Background`, `Toybox.Activity`
- **[확인 완료 ✅] 알림 기능** — `Toybox.Notifications.showNotification()`으로 즉시 알림 표시 가능. 단, 예약 발송 기능은 없음
  - 구현 방법: 밥줄 때마다 `Background.registerForTemporalEvent(48시간 후)`로 재등록 → 시간 됐을 때 백그라운드에서 마지막 급식 시각 확인 → 조건 충족 시 `showNotification()` 호출
  - `Notifications` 퍼미션 선언 필요 (manifest.xml)
  - 주의: 앱 종류에 따라 지원 기기가 다를 수 있음. Forerunner 255 지원 여부 실기기 확인 필요

- **[확인 완료 ✅] 워치페이스 연동 불가** — Connect IQ 앱은 각자 독립된 Storage를 가지며 앱 간 데이터 공유 없음
  - 워치페이스에서 장치 앱의 Storage를 읽는 것 불가
  - **결론**: 장치 앱 단독으로 구현하는 현재 방향이 맞음. 워치페이스 연동은 고려 불필요
  - **대안으로 고려 가능**: Glance(글랜스) — 버튼 한 번으로 접근하는 미니 뷰. 별도 앱이지만 장치 앱과 함께 번들 배포 가능한지 추가 확인 필요 (v2 검토 항목)

## 라이브러리 참고

**Connect IQ API 문서**: https://developer.garmin.com/connect-iq/api-docs/

| 모듈 | 용도 |
|------|------|
| `Toybox.Background` | 백그라운드 서비스 등록 및 이벤트 처리 |
| `Toybox.Activity` | 런 중 거리 등 활동 데이터 읽기 |
| `Toybox.ActivityMonitor` | 일별 활동 히스토리, 걸음 수, 심박수 |
| `Toybox.Application.Storage` | 토큰·상태 영구 저장 |
| `Toybox.Notifications` | 배고픔 알림 (지원 여부 추가 확인 필요) |
| `Toybox.Timer` | 시간 기반 성장 처리 |
| `Toybox.WatchUi` | 화면 렌더링, 버튼 입력(밥주기) |

### 일반 제약

- Connect IQ 내부 저장소 용량 제한 (스프라이트 파일 크기 주의)
- 배터리 절약 고려 (백그라운드 동작 최소화)
- v1은 Forerunner 255 단독, v2에서 멀티 기기 확장
